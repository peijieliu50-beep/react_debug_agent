# PyTorch 训练故障排查手册

## 1. Loss 不下降 / 不收敛
常见原因与排查顺序：
- 学习率不合适：过大导致 loss 震荡或发散，过小导致几乎不动。建议从 1e-3 起，按 10 倍量级调。
- 忘记调用 optimizer.zero_grad()：梯度累积导致更新异常。标准三步：zero_grad() → loss.backward() → optimizer.step()。
- 数据未归一化：图像应归一化到 [0,1] 或标准化（如 transforms.Normalize）。
- 标签错误：分类任务标签需为从 0 开始的整数 LongTensor。
- 模型最后一层与损失函数不匹配：用 CrossEntropyLoss 时网络最后不要再加 softmax。
- 模型容量不足或数据太难：适当加深/加宽网络。

## 2. Loss 变成 NaN / Inf（训练发散）
- 学习率过大：先降低 10 倍。
- 出现 log(0) 或除零：检查自定义 loss、加 eps（如 1e-8）。
- 梯度爆炸：使用梯度裁剪 torch.nn.utils.clip_grad_norm_(model.parameters(), max_norm)。
- 输入数据含 NaN：检查数据预处理。

## 3. 梯度消失
- 激活函数饱和：用 ReLU / LeakyReLU 替代 sigmoid/tanh。
- 网络过深无残差：引入 BatchNorm 或残差连接。
- 初始化不当：使用 Kaiming/Xavier 初始化。

## 4. 准确率不提升但 loss 下降
- 评估时未切换 model.eval()，BatchNorm/Dropout 行为不一致。
- 训练/测试数据分布不一致或数据泄漏。
- 类别不平衡：使用加权损失或重采样。
